## Virtual Environment 

 Virtual Environments (venv)
Let me guess — you have no idea what this is. That's exactly why I'm writing this.
If you choose Python, you need to know about virtual environments before you install anything.
I wish someone had told me this earlier. It would have saved me a lot of time and headaches.
What is a virtual environment?
A virtual environment is an isolated space for your project. Every project gets its own packages, its own versions, and its own dependencies — completely separate from everything else on your computer.
Without it, all your projects share the same packages globally. This causes version conflicts and messy setups that are hard to fix.
How to create one
bash# Create
python -m venv venv

# Activate (Windows)
venv\Scripts\activate

# Activate (Mac/Linux)
source venv/bin/activate

# Install packages (always after activating!)
pip install fastapi

# Save your packages
pip freeze > requirements.txt

# Install from requirements (on a new machine)
pip install -r requirements.txt
The golden rule

Always activate your virtual environment before doing anything in your project.
